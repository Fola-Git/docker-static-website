Welcome to the Docker and NGINX Tutorial

Learn how to deploy a static website using Docker and NGINX

What is Docker?

Docker is an open platform for developing, shipping, and running applications. It allows you to package applications and their dependencies into a container, which can be run consistently across different environments.

A Docker image is a template used to create containers. It contains everything needed to run an application: code, runtime, libraries, environment variables, and configurations. A Docker container is a running instance of an image.

In this tutorial, we'll use Docker to run a static website served by NGINX.

What is NGINX?

NGINX is a high-performance web server that can also be used as a reverse proxy, load balancer, and HTTP cache. In our case, we'll use NGINX to serve static files (HTML, CSS, JavaScript) for our website.

What is a Dockerfile?

A Dockerfile is a text document that contains instructions to build a Docker image. It specifies the base image, copies application files, installs dependencies, and sets up the environment for the application.

In our example, the Dockerfile uses the official NGINX image and copies our website files into the appropriate directory for NGINX to serve.

Step-by-Step: Creating and Running a Docker Image

  1. Create the project directory: Use the following command to create a directory where your files will be stored: 
    mkdir static-website
  2. Create an HTML file: Inside the directory, create an index.html file. This will be the main webpage. Add the HTML content you want to display. 
    nano index.html
  3. Create a Dockerfile: Inside the same directory, create a Dockerfile with the following content: 
    nano Dockerfile
    
FROM nginx:alpine
COPY . /usr/share/nginx/html
  4. Build the Docker image: In the terminal, run the following command to build the Docker image: 
    docker build -t my-static-website .
    This command creates a Docker image named my-static-website.
  5. Run the Docker container: After building the image, run it in a container using: 
    docker run -d -p 8080:80 my-static-website
    This command maps port 8080 on your local machine to port 80 inside the container.
  6. Access the website: Open your browser and go to: 
    http://localhost:8080
    You should see the website you created!

Understanding the Dockerfile

Let's break down the content of the Dockerfile:

  • FROM nginx:alpine: This tells Docker to use the official NGINX image with the Alpine Linux distribution. Alpine is a lightweight version of Linux, making the image smaller and faster to download and run.
  • COPY . /usr/share/nginx/html: This copies all the files from the current directory (the one containing the Dockerfile and index.html) into the /usr/share/nginx/html directory inside the container. This is the directory NGINX uses to serve static files.
